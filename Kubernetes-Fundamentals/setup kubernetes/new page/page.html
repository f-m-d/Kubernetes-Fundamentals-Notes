Kubernetes Setup - Introduction and Minikube 




Vi sono diversi modi per settare Kubernetes:

Locally:
Tool come Minikube, Microk8s, Kubeadm.
Sono utili per i developer o per chi vuole imparare Kubernetes.
Kubeadm è usare per bootstrappare e gestire production grid,
generalmente per gestire un multi-node setup.


Cloud:
E' possibile effettuare soluzioni cloud per Kubernetes,
come Google Cloud Platform, Amazon Web Services (AWS), Microsoft Azure, IBM Clouds etc.


Altro:
Nel nostro caso di corso, possiamo accedere ad un cluster già abilitato
su www.kodekloud.com o play-with-k8s.com


Minikube



Un esempio di installazione che facciamo è con Minikube.
Generalmente, impiega molto tempo installare tutte le componenti
di Kubernetes su una macchina (etcd, API Server etc.)

Minikube semplicemente unisce tutte le componenti
di un nodo cluster di Kubernetes in una singola immagine,
creando un singolo nodo per gestire un cluster.

L'intero bundle è disponibile come un ISO image
ed è disponibile online per il download.

Minikube provvede una executable command line utility,
capace di scaricare automaticamente la ISO e di deployarla
in una Virtualization Platform (VMWare, VirtualBox)
NB: Devi avere Hyper-V abilitato sulla macchina per virtualizzare!

Infine, per interagire col Kubernet cluster, necessiti
della command line tool kubectl.


In sintesi, necessiti di 3 cose:
  • Hyper-V (per la virtualizzazione)
  • Kubectl
  • Minikube
